## Developer Logs

Polishing frontend milestone 2.

We're going to change directions a bit for the app state manager and profile.
Move the app state manager into the router folder, because it will only be used in conjunction with the router.
Delete all code in it relating to the profile subscription.
This is the new plan for this:
The profile feature will have two pages: /setup and /profile. They will share widgets. /setup is the blocker page for if the user hasn't set their username yet.
Instead of updating the profiles table, these pages update the auth user metadata with the supabase CLI auth feature.
In the database there will be a trigger that copies the auth user metadata to update the profiles table.
(We'll also make sure that on onboarding a profile is created when a user is created, maybe this will need a trigger as well.) 
Now the app state manager will check the JWT auth metadata instead of the profile table for the username. To do this, it doesn't require a database call or take any more time than it already does to check if the user is authenticated.
